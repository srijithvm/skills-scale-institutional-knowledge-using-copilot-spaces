# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

### Interaction Points
- **With Product Managers**: Discuss feature requirements, acceptance criteria, and trade-offs
- **With Project Managers**: Participate in planning, status updates, and risk identification
- **With Quality Assurance Lead**: Coordinate on test plans, quality standards, and issue resolution
- **With Change Manager**: Notify of code deployments and coordinate breaking changes
- **With UX Researcher**: Review user research findings to inform implementation
- **With Release Coordinator**: Coordinate code freezes, deployments, and rollback plans

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

### Interaction Points
- **With Developers**: Define requirements, review implementation, validate solutions
- **With Project Managers**: Align on timelines, scope, and delivery priorities
- **With UX Researcher**: Commission research, incorporate findings into roadmap
- **With Quality Assurance Lead**: Define acceptance criteria and quality standards
- **With Change Manager**: Communicate product changes to stakeholders
- **With Release Coordinator**: Coordinate feature releases and rollout timing

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

### Interaction Points
- **With Developers**: Monitor progress, identify blockers, manage capacity
- **With Product Managers**: Align on scope, timeline, and priorities
- **With Quality Assurance Lead**: Plan testing phases, manage quality gates
- **With Change Manager**: Coordinate change impact analysis and communication
- **With UX Researcher**: Schedule user research, incorporate findings into timeline
- **With Release Coordinator**: Plan release timeline, coordinate pre-release activities

---

## Quality Assurance Lead

### Role Summary
The Quality Assurance Lead establishes quality standards, coordinates testing activities, and ensures all deliverables meet defined quality and compliance requirements before release.

### Responsibilities
- Define quality standards and acceptance criteria
- Plan and coordinate testing activities (unit, integration, UAT, regression)
- Identify, track, and prioritize quality issues
- Review deliverables against quality gates
- Ensure compliance with regulatory and organizational standards
- Advocate for quality and prevent premature releases

### Goals
- Ensure all deliverables meet quality standards
- Reduce defects in production
- Build confidence in release readiness
- Continuously improve quality processes

### Typical Communication
- Quality status reports and risk assessments
- Test plan reviews and walkthroughs
- Issue tracking and escalations
- Quality gates reviews before release

### Interaction Points
- **With Developers**: Review code quality, coordinate on bug fixes, validate test coverage
- **With Product Managers**: Clarify acceptance criteria, validate feature completion
- **With Project Managers**: Report quality status, identify risks to timeline
- **With Change Manager**: Assess impact of changes on quality
- **With UX Researcher**: Ensure user experience meets quality expectations
- **With Release Coordinator**: Perform pre-release quality gate checks

---

## Change Manager

### Role Summary
The Change Manager oversees all change requests, manages impact analysis, and ensures stakeholders are informed and prepared for project changes. This includes scope changes, timeline adjustments, and deployment changes.

### Responsibilities
- Manage change request intake and evaluation
- Perform change impact analysis (timeline, budget, quality, resources)
- Communicate changes to all affected stakeholders
- Track change history and maintain change register
- Coordinate stakeholder approval and sign-off
- Plan change implementation and rollback strategies

### Goals
- Minimize disruption from unexpected changes
- Ensure transparency and buy-in on changes
- Maintain project baselines and control
- Prevent scope creep and hidden costs

### Typical Communication
- Change requests and impact assessments
- Stakeholder change notifications
- Change logs and decision records
- Escalation notifications for major changes

### Interaction Points
- **With Developers**: Understand technical impact of proposed changes
- **With Product Managers**: Evaluate prioritization and business impact
- **With Project Managers**: Assess impact on timeline and resources
- **With Quality Assurance Lead**: Identify testing impact and quality risks
- **With UX Researcher**: Assess impact on user experience
- **With Release Coordinator**: Plan change deployment and communication

---

## UX Researcher

### Role Summary
The UX Researcher brings user insights and perspective to all project decisions. They conduct research, validate design assumptions, and ensure the project delivers genuine user value and usability.

### Responsibilities
- Plan and conduct user research (interviews, surveys, usability testing, analytics review)
- Share research findings with product, design, and engineering teams
- Validate design concepts and prototypes with users
- Advocate for user needs in decision-making
- Define and track user experience metrics

### Goals
- Ensure solutions meet real user needs
- Improve user satisfaction and adoption
- Reduce rework due to poor UX decisions
- Build empathy for users across the team

### Typical Communication
- Research findings and recommendations
- User personas and journey maps
- Usability test reports and recommendations
- User feedback and metrics dashboards

### Interaction Points
- **With Developers**: Share user insights to inform design and implementation
- **With Product Managers**: Commission research, collaborate on roadmap prioritization
- **With Project Managers**: Coordinate research timeline, report on user satisfaction
- **With Quality Assurance Lead**: Validate user experience meets quality expectations
- **With Change Manager**: Assess user impact of proposed changes
- **With Release Coordinator**: Gather user feedback post-release

---

## Release Coordinator

### Role Summary
The Release Coordinator plans, orchestrates, and manages all release activities. They ensure releases are well-coordinated, properly documented, and successfully deployed while minimizing risk and production issues.

### Responsibilities
- Create and maintain release plans and schedules
- Coordinate code freezes, builds, and deployment activities
- Manage release documentation and release notes
- Coordinate pre-release testing and quality gates
- Execute deployments and monitor for issues
- Plan and execute rollback procedures if needed
- Communicate release status to stakeholders
- Document lessons learned for future releases

### Goals
- Execute releases with zero or minimal production issues
- Minimize unplanned downtime
- Maintain transparency on release status
- Continuously improve release processes

### Typical Communication
- Release schedules and timelines
- Pre-release checklists and sign-offs
- Deployment logs and status updates
- Post-release incident reports

### Interaction Points
- **With Developers**: Coordinate code submissions, manage deployment timing
- **With Product Managers**: Align on feature release timing and rollout strategy
- **With Project Managers**: Coordinate with project timeline, manage dependencies
- **With Quality Assurance Lead**: Ensure quality gates are met before deployment
- **With Change Manager**: Notify of deployment changes, coordinate stakeholder communication
- **With UX Researcher**: Gather user feedback post-release
